# Rewrite Report Playbook

本文件展开 Stage 5 中 `rewrite-report.md` 的生成规则。全局约束以 `SKILL.md` 为准；本文件只补转写报告的固定结构、混合级参照规则和失败处理。

## 适用范围

- 报告文件固定为 `.paper-condenser-tmp/<document-slug>/rewrite-report.md`。
- 这是 Stage 5 的正式伴随输出，不是第五个中间工件。
- 报告面向人类用户继续修改稿件，因此首版只要求 Markdown，不要求 JSON 镜像。

## 固定结构

### 1. Run Summary

- 记录源文件、目标设置、模板选择、最终输出路径。

### 2. Stage Decisions

- 只摘要真正影响最终稿的 Stage 1-4 决策。
- 不重复转抄全部工件内容。

### 3. Final Draft Section Map

- 对最终稿的每个章节或小节都建立章节级映射。
- 每条映射至少说明：
  - 最终稿部分
  - 对应的原文范围或原文章节
  - 主要采用了哪些原始论点、数据、段落群或 supporting elements
  - 是否发生明显压缩、合并或重组

### 4. Key Paragraph And Element Notes

- 只对关键段落、关键图表、关键表格、关键引用做更细说明。
- 以下情况必须进入本节：
  - 一个段落跨多个原文章节重组
  - 一个段落承载核心结论或主要创新
  - 一个关键图表或关键引用明显影响最终论证
  - 某部分只能占位，尚未完全精修

### 5. Unresolved Risks / Follow-up

- 记录尚未完全精修的地方。
- 记录用户后续修改时最需要回看的原文位置和原因。

## 混合级参照规则

- 所有最终稿章节/小节必须有章节级参照。
- 不要求每个段落都建映射。
- 只有关键段落和关键元素才进入细粒度说明。
- 报告必须可追踪，但不能膨胀成逐段逐句比对清单。

## 常见失败场景

### 只有最终稿，没有报告

- 表现：
  - `final-draft.tex` 已生成，但没有 `rewrite-report.md`
- 处理：
  - Stage 5 不算完成
  - 回到报告生成步骤补齐输出

### 报告只有流水账，没有参照

- 表现：
  - 报告只总结“做了什么”，但没有指出最终稿各部分大致来自原文哪里
- 处理：
  - 回到 `Final Draft Section Map`
  - 为每个章节/小节补齐章节级原文参照

### 报告过细，退化成逐段对照表

- 表现：
  - 报告为所有段落都建立细粒度映射，噪声过大
- 处理：
  - 保留章节级全覆盖
  - 只对关键段落和关键元素保留细化说明
